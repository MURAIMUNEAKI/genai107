ご提示いただいた「地域未来交付金 デジタル実装型 TYPE A/V/S 制度概要（令和7年12月24日）」の資料に基づき、申請書作成に直結する詳細な要件とポイントを4つのセクションに分けてまとめました。

各セクションは、申請書の記述に必要な情報を網羅するように構成しています。

# 1. 共通要件および制度全体概要

本交付金は、デジタル技術を活用して地域の課題解決や魅力向上を図る地方公共団体の取組を支援するものです。申請にあたり、すべてのTYPEに共通して求められる基本的かつ絶対的な要件です。

### 【制度の目的と基本姿勢】
*   **目的:** 地方の伸び代と特性を活かし、東京一極集中の是正、地場産業の付加価値向上、地方の暮らしの安定（「強い経済」の構築）を実現すること。
*   **対象:** 都道府県、市区町村、一部事務組合、広域連合（港務局含む）。
*   **原則:** 「構造改革のためのデジタル原則」に沿った事業であること。アナログ規制の見直しや、デジタル完結を前提としたプロセス設計が求められます。

### 【必須共通要件（絶対遵守事項）】
以下の要件を満たしていない場合、審査の土俵に乗りません。

1.  **デジタル活用の目的の明確化**
    *   単なるシステム導入ではなく、「地域の課題解決」や「魅力向上」に資するものであることが論理的に説明されている必要があります。
    *   **KPIの設定:** 事業の成果を客観的に測定するため、「アウトプット指標（活動量）」と「アウトカム指標（住民への裨益・効果）」の両方を設定し、複数年（3カ年）にわたって計測・公表する必要があります。
        *   *既存事業の拡充の場合:* 過去の実績値に上乗せした目標設定が必須です。

2.  **推進体制の確立（コンソーシアム等）**
    *   自治体単独ではなく、地域内外の関係者と連携する体制が必要です。
    *   **構成員:** 地方公共団体、民間事業者、地域団体、国、専門家など。
    *   **役割分担:** サービスの実装、モニタリング、改善において、誰が何を担うかを明確にします。利用者の参画も推奨されています。

3.  **持続可能性（サステナビリティ）**
    *   **実装期間:** 交付決定日から令和9年（2027年）3月31日まで。
    *   **事業終了後:** 実施計画期間（実装1年、運営2年）終了後も、地域住民に対して継続的にサービスを提供する義務があります。「実証実験で終わり」は認められません。
    *   **財源:** 交付金終了後のランニングコストは、自治体が確保する必要があります（歳入計画等の提示が必要）。

4.  **調達における公正性**
    *   原則として**「一般競争入札」**を実施する必要があります。
    *   特定の事業者（スタートアップ等）を指名する場合でも、十分な競争原理が働く環境（複数見積もり等）や、随意契約の正当な理由（トライアル発注制度の活用など）の説明が必須です。

5.  **他の補助金との重複排除**
    *   国による他の補助金の対象となる事業は、そちらを優先利用する必要があります。本交付金は「他でカバーできない部分」や「独自の組み合わせ」に対する支援です。

### 【申請上限数と事業費】
*   **申請上限（TYPE A/V合算）:**
    *   都道府県：最大9事業
    *   市区町村：最大5事業
    *   ※TYPE Sはこのカウント枠外です。
    *   ※TYPE Aの「地域間連携事業」やTYPE Vの「従たる申請」については枠外措置があります。
*   **事業費要件:** 交付対象事業費が100万円を超える事業であること。
*   **地方負担:** 地方交付税の増額交付等で対応（裏負担への財政措置あり）。

### 【対象外経費・NG事例】
*   **庁内業務効率化のみ:** 住民に直接裨益しない、職員の勤怠管理システムや単なるPC整備等は対象外です。
*   **単発イベント:** 継続性のないイベントや、一時的な講習会等は対象外です。
*   **事前着手:** 交付決定日より前の契約や発注は一切認められません。

# 2. デジタル実装型：TYPE A（地域住民等利用推進型）

従来の「TYPE 1」を刷新した区分です。「住民が使って効果を実感できる」ことに重点が置かれています。

### 【概要と補助率】
*   **目的:** 地域住民等がデジタルサービスを利用し、その効果を実感できる取組。
*   **補助率:** 1/2
*   **上限額:** 国費1億円（事業費ベース2億円）
*   **キーワード:** 「実装」「住民裨益」「継続利用」

### 【詳細審査要件（No.1〜9）】
申請書では以下の全項目を満たす旨を証明する必要があります。

1.  **課題解決・魅力向上:** どのような地域課題に対し、デジタルがどう寄与するか具体的に記載。
2.  **地域・暮らしへの実装:** 庁内のみならず、地域社会や住民生活の中にサービスが組み込まれること。
3.  **【重要】住民への直接裨益:**
    *   住民が直接利用するサービス（アプリ等）、または職員と住民が共に利用することで住民にメリットがあるサービスであること。
    *   *注意:* 行政事務の効率化による「処理時間の短縮」だけでは、TYPE Aの住民裨益とは認められません。
4.  **KPI設定:** 複数年度にわたる計測が可能で、客観的かつ妥当な水準であること。
5.  **実装期間:** 令和9年3月31日までに確実に実装完了すること。
6.  **継続性:** イベント等の単発ではなく、恒久的なサービスであること。
7.  **優良モデルの活用:** 他地域で実績のあるモデルや、実証済みのサービスを活用すること（ゼロからのスクラッチ開発はリスクが高いため推奨されにくい）。
8.  **推進体制:** コンソーシアム等の確立。
9.  **事業費規模:** 100万円以上。

### 【優遇措置（加点項目）】
TYPE Aでは、以下の要素を取り入れることで審査時の評価が高くなります。該当数が多いほど高評価です。

1.  **モデル仕様書の活用:**
    *   デジタル庁が公開する「デジタル地方創生モデル仕様書（23類型）」に準拠したシステムを導入する場合。
    *   *必須:* 仕様書の必須機能を削除せず実装すること（追加は可）。
    *   *対象例:* AIオンデマンド交通、母子手帳アプリ、オンライン診療、書かない窓口、防災システム等。
2.  **共通SaaS等の活用:**
    *   「デジタル地方創生サービスカタログ」に掲載されているSaaS等を選定する場合。安価で迅速な導入が評価されます。
3.  **DMP（デジタルマーケットプレイス）の活用:**
    *   デジタル庁のDMPカタログサイトを通じて製品選定・調達を行う場合。
4.  **自治体窓口DXSaaSの利用:**
    *   デジタル庁提供のガバメントクラウド上の「書かないワンストップ窓口」サービスを利用する場合（窓口BPRの実施が前提）。
5.  **マイナンバーカードの利活用:**
    *   サービスの認証や機能にマイナンバーカードを活用する場合（避難所受付、図書館カード、公共施設予約など）。
6.  **スタートアップの活用:**
    *   設立15年以内、未上場等の要件を満たすスタートアップ企業がサービスの提供主体（または一部機能提供）となる場合。
    *   *注意:* 単なる下請けやコンサルではなく、プロダクトの実装が必要です。
7.  **地域間連携の促進:**
    *   複数の自治体が連携して同一・類似の課題解決に取り組む場合。
    *   *枠外措置:* 「一定の地域間連携（定住自立圏など）」の場合は申請枠外カウントとなり、有利です。
8.  **未採択団体への優遇:**
    *   過去にTYPE1等で採択実績がない団体は優先採択されます。
9.  **新たな分野への挑戦:**
    *   既採択団体でも、過去に取り組んでいない新分野（例：防災で採択済みだが、次は医療に取り組む等）の場合は加点。

### 【申請書の記述ポイント】
*   **具体性:** 「誰が」「いつ」「どのようなシーンで」使うのか、利用シーン（ユースケース）を明確に描くこと。
*   **サービスの特定:** 「何かAIを使います」ではなく、「〇〇社の××システム相当」や「モデル仕様書の〇〇類型」と具体的に特定すること（入札前であっても想定を示す）。
*   **住民の巻き込み:** どのように周知し、利用率（ダウンロード数やアクティブユーザー数）を上げるかのプロモーション計画を具体的に書くこと。

# 3. デジタル実装型：TYPE V（先進的デジタル公共財活用型）

複数の自治体が連携し、国が推奨する「デジタル公共財」を活用して社会課題解決に挑む、高難易度かつ大規模な類型です。

### 【概要と補助率】
*   **目的:** 複数団体による「共同調達・共同利用」と「デジタル公共財」の活用。
*   **補助率:** 2/3（TYPE Aより高い）
*   **上限額:** 国費4億円（事業費ベース6億円）
*   **申請条件:** 1団体あたり1申請のみ（従たる申請は追加1件可）。必ず「事前相談」が必要です。

### 【対象となる「デジタル公共財」の定義】
以下のいずれかを活用することが必須要件です。

1.  **デジタル公共財:**
    *   デジタル庁が推奨するシステム（データ連携基盤、JPKI、マイキープラットフォーム等）。
    *   特に「エリアデータ連携基盤」の活用が強く推奨されます。
2.  **新興型デジタル公共財:**
    *   将来的に公共財となりうる先進技術。
    *   **高度なAI活用:** 単なる文字起こし等は不可。ロジック設計にAIが深く関与し、効率化と品質向上を両立するもの（例：AIによる救急搬送レコメンド）。
    *   **新規性の高いマイナンバーカード活用:** 教育データの連携、子ども・保護者の紐づけ管理など。
    *   **Web3.0/ブロックチェーン:** NFTやDAOを活用した関係人口創出、地域通貨など。

### 【必須要件（モデル性評価）】
TYPE Vは「事業性」に加え、「モデル性（先進性）」が50点満点で厳しく審査されます。

1.  **共同調達・共同利用の実態:**
    *   単に同じベンダーを使うだけでは不十分です。「仕様の共通化」「システムの共同運用」「データ連携基盤の共有」など、実質的な連携が必要です。
    *   *パターン:* 代表団体が一括契約し負担金を徴収する、またはデータ連携基盤を共有しアプリは個別契約する、など。
2.  **データ連携基盤の活用（加点大）:**
    *   エリアデータ連携基盤（FIWARE Orion等、推奨モジュール採用）を導入し、複数サービス間でデータを連携させること。
    *   パーソナルデータと非パーソナルデータの両方を扱うと高評価。
3.  **広域連携の規模:**
    *   都道府県内の全自治体が参加するなど、規模が大きいほど加点されます（最大10点）。
4.  **相互運用性の確保:**
    *   GIF（政府相互運用性フレームワーク）への準拠、オープンAPIの活用。
5.  **セキュリティ:**
    *   スマートシティセキュリティガイドラインへの準拠。

### 【申請書の記述ポイント】
*   **連携の必然性:** なぜ単独ではなく共同で行うのか（コスト削減、データ流通のメリット）を定量的・定性的に説明すること。
*   **データフロー:** データ連携基盤を中心に、どのデータがどう流れ、どうサービスに活用されるか、アーキテクチャ図を用いて説明すること。
*   **KPIの高度化:** 単なる利用者数だけでなく、Well-Being指標やロジックツリーを活用し、政策目的達成度を測る指標を設定すること。
*   **将来の横展開:** このモデルが全国の他地域でも再現可能であることをアピールすること。

# 4. デジタル実装型：TYPE S（デジタル行財政改革特化型）

国の「デジタル行財政改革」と直結する、極めて先導的なトップランナー向けの類型です。採択予定数は全国で6件程度と非常に狭き門です。

### 【概要と補助率】
*   **目的:** 国や地方の統一的・標準的なデジタル基盤への横展開につながる、先行モデル的な取組。
*   **補助率:** 3/4（最も高い）
*   **上限額:** 国費2.25億円（事業費3億円）＋別途、標準仕様策定等の委託調査費（国費100%）が最大1億円程度つく可能性があります。
*   **枠外措置:** デジタル実装型の申請上限数（9事業/5事業）にはカウントされません。

### 【対象分野】
デジタル行財政改革会議事務局が指定する「主な改革分野」に合致する必要があります。
*   **教育:** 校務支援、AIドリル等。
*   **子育て・福祉:** 保育所業務、児童相談等。
*   **医療・介護:** 医療DX、救急搬送、介護認定等。
*   **防災:** 避難所運営、被災者台帳等。
*   **モビリティ:** 自動運転、MaaS（ライドシェア含む）。
*   **インフラ:** 道路・水道等の維持管理。

### 【審査の視点と要件】
1.  **国・事務局のコミットメント:**
    *   単なる一自治体の改善ではなく、将来的に「国の標準仕様」や「制度改正」につながるポテンシャルがあるか。デジタル行財政改革事務局が「一緒にやる価値がある」と認める必要があります。
2.  **EBPMと利用者起点:**
    *   「なんとなく便利」では不可。EBPM（証拠に基づく政策立案）の手法を用い、データを収集・分析し、施策の効果を定量的に証明できる設計になっていること。
    *   サービスデザイン思考に基づき、徹底的に利用者（住民）目線で設計されていること。
3.  **行財政改革効果:**
    *   利便性向上だけでなく、具体的に「業務時間が〇〇時間削減される」「財政支出が〇〇円削減される」といった行財政改革の成果が見込まれること。
4.  **標準化への貢献:**
    *   この取組の成果（仕様書、データモデル、運用フロー）を、他自治体がそのまま使えるレベルに標準化する意思があること。

### 【実施体制とプロセス】
*   **伴走支援:** 採択後は、国が委託するコンサルタント等が入り、標準仕様策定や効果測定の支援を行います（これに協力することが義務）。
*   **有識者審査:** 通常の審査に加え、デジタル行財政改革会議事務局の各分野担当チームや有識者による厳格な審査が行われます。

### 【申請書の記述ポイント】
*   **圧倒的な先導性:** 既存の優良事例のコピーではなく、「全国初」「制度の壁を突破する」ようなチャレンジ要素が必要です。
*   **分析の深さ:** 現状の課題（コスト構造、業務フローのボトルネック）をデータで精緻に分析し、改革後の姿（Before/After）を定量的に示してください。
*   **覚悟:** 庁内の業務フローを抜本的に変える、条例改正も辞さないといった、首長レベルの強いコミットメントを示す書きぶりが求められます。
