TYPE S
デジタル行財政改革特化型
詳細解説

補助率3/4。全国でわずか6件程度のトップランナー枠。
「国の仕組み」そのものを作り変える、構造改革と標準化のパイオニアを募集します。

↓
Concept

基本概念と戦略的意義

TYPE Sの定義
単なる地域課題解決ではありません。「国の制度やシステムの標準を作る」という極めて高い視座が求められます。
主眼は地域実装ではなく、「構造改革 (Reform)」と「標準化 (Standardization)」です。
成功事例を「標準仕様」として全国に展開する意思が必要です。
Premium

予算規模と破格の待遇

最高補助率
補助率 3/4
上限2.25億円 (事業費3億円)。
追加支援 (+1億円)
別途、国の調査事業としてコンサルタントや専門家チームが派遣されます。
実質最大4億円規模のプロジェクトとなります。
申請枠外
TYPE A/Vの申請上限数にカウントされません。
採択予定数：全国で計6件程度（狭き門）。
Fields

対象となる改革分野

デジタル行財政改革会議事務局が指定する以下の分野。

教育
校務支援、AIドリル、不登校支援。
子育て・福祉
保育所業務DX、児相DX、こどもデータ連携。
医療・介護
医療DX、救急搬送、介護認定AI。
防災・インフラ
避難所DX、被災者台帳、インフラAI点検、モビリティ(MaaS, ライドシェア)。
Requirements

必須要件（3つのハードル）

1. 先導性とコミット
「便利になる」だけでなく、「公務員の工数削減」や「財政改革」に繋がること。
国がコミットする価値がある改革であること。
2. 利用者起点とEBPM
サービスデザイン思考に基づく設計。
ロジックモデルを用いた、科学的な成果指標（KPI）の設定と検証。
3. 強力な推進体制
首長・幹部のコミット、庁内縦割りの打破、外部専門家との連携体制。
Process

審査と採択後の動き

審査基準
有識者審査およびオンラインヒアリング審査が必須。
「社会変革性」「標準化の可能性」「EBPMの妥当性」が問われます。加点措置はありません。
採択後の義務
国の委託事業者と共に「標準仕様書（導入ガイドブック）」を作成し、成果をオープンデータ化して他団体への横展開に協力する義務があります。
「お金だけもらってあとは自分たちで」は通用しません。
Strategy

申請書作成の戦略

課題の深掘り
「窓口が混む」レベルは不可。「年間〇時間の工数、〇円のコスト、手戻り〇％」と定量的に構造解析する。
ROI（投資対効果）
システム費に対して、何年でペイするか。具体的な「行政コスト（人件費等）削減額」を打ち出す。
解決策の革新性
既存パッケージ導入は不可。技術的・制度的ブレイクスルーを含む提案が必要。
