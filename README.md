# 源内 AI アプリ

## 概要

源内（GenAI）は、デジタル庁が開発・運用する生成 AI 利活用基盤です。行政職員が業務特化の生成 AI アプリケーションを、迅速かつ安全かつ簡単に利用できる環境を提供します。  
源内は大きく分けて2種類のシステムで構成されます。

- [源内 Web](https://github.com/digital-go-jp/genai-web): 利用者が直接さわる Web アプリケーション
- [行政実務用 AI アプリ](https://github.com/digital-go-jp/genai-ai-api): 生成 AI を活用したマイクロサービス

本リポジトリではこのうち、中央省庁で実際に展開されている行政実務用 AI アプリの一部を公開しています。  
源内の取り組み・構想・生成 AI 活用方法の詳細等は、[デジタル庁のTechブログ](https://digital-gov.note.jp/m/m90208c3610d0)で発信中です。

## 機能一覧（源内AI Web v1.0.7）

本パッケージ（`index.htm` / `app/` / `app2/`）で利用できる Web アプリの一覧です。  
PHP が動作する Web サーバーに配置し、`api/.env` に API キーを設定して利用します。

### セットアップ（Web アプリ）

1. リポジトリを PHP 対応の Web サーバーに配置する
2. `api/.env` を作成し、利用する機能に応じてキーを設定する

```env
# 必須（多くのアプリで使用）
GEMINI_API_KEY=your-gemini-api-key
GENNAI_PASSWORD=kaiin

# チャットで OpenAI を使う場合
OPENAI_API_KEY=your-openai-api-key

# 議会答弁（Dify アプリ連携）
DIFY_ANSWER_KEY=your-dify-api-key

# 任意（モデル指定）
GEMINI_MODEL=gemini-3.1-flash-lite
```

3. ブラウザで `index.htm` を開き、`.env` の `GENNAI_PASSWORD` でログインする
4. `app/main.htm` から各アプリを利用する

### 注意事項（全アプリ共通）

- AI の出力は必ずしも正確ではありません。重要な判断・公表前には原典・担当部署での確認が必要です
- 機密性の高い情報（機密性3情報等）の入力は避けてください
- `api/.env` には API キーを含めます。公開リポジトリへのコミットや Web からの直接参照はしないでください（`.htaccess` で `.env` へのアクセスを拒否）

## 行政事業レビューAI 利用マニュアル

事業の資料（PDF/Word/CSV/TXT）をアップロードするだけで、行政事業レビューに必要な観点を AI が整理・採点し、改善提案までを含む 5 部構成のレポートを自動生成するアプリです（`app2/review.htm`）。  
予算編成期の事前点検、レビューシートの下書き、幹部説明用の要約づくりなどを想定しています。

### 主な機能

| 機能 | 内容 |
|------|------|
| 複数ファイルアップロード | PDF / DOCX / CSV / TXT に対応（各 20MB まで・複数選択可）。クリック選択またはドラッグ＆ドロップ。アップロード状況・解除をファイルごとに表示 |
| 基本情報の入力 | 事業名（必須）・年度・所管課を入力すると、レポートのヘッダーに反映 |
| 5部構成レポート生成 | ①幹部向け要約 ②レビューシート ③観点別採点（100点満点） ④改善提案（短期/中期/構造見直し＋TOP3） ⑤追加確認事項・根拠箇所 |
| 観点別採点 | 必要性 / 有効性 / 効率性 / 執行適正 / 透明性・KPI妥当性 を各 20 点（計 100 点）で採点し、総合判定（現状通り〜廃止検討）を提示 |
| ストリーミング表示 | 生成結果をリアルタイムに Markdown 表示 |
| Word テンプレート | 令和7・8・9年度のレビューシート Word テンプレートをダウンロード可能 |
| サンプルデータ | 3 種類のサンプル事業資料（観光インバウンド・後継者育成・パンフレット印刷）をダウンロードして試用可能 |
| エクスポート | 生成結果を「コピー」「テキスト保存（.txt）」「Word出力（.docx）」 |
| 利用履歴 | ブラウザ内 IndexedDB に保存（7日保持）。`?id=` 付き URL で過去結果を復元 |

### 使い方

1. `app/main.htm` または `app/apps.htm` のカード一覧から **行政事業レビューAI** を開く（直接 `app2/review.htm` を開いてもよい）
2. **事業名**（必須）を入力し、必要に応じて **年度** ・ **所管課** を入力する
3. **対象資料ファイル** をアップロードする
   - 対応形式: PDF / DOCX / CSV / TXT（各 20MB まで・複数可）
   - 事業概要書・予算/決算資料・成果報告書・KPI 資料などをまとめてアップロードできます
   - 手元に資料がない場合は、画面上部の **サンプルデータ** を使って動作を確認できます
4. すべてのファイルが「準備完了」になったら **レビューを実行** を押す
5. 生成されたレポートを確認し、必要に応じて **コピー / テキスト保存 / Word出力** で持ち出す
   - レビューシートに転記する場合は、**Wordテンプレート**（令和7〜9年度）と併用すると便利です

### レポートの構成（出力イメージ）

```
# 1. 幹部向け要約        … 結論（1行）／要点／幹部が判断すべき論点（3点以内）
# 2. レビューシート       … 基本情報／事業概要／予算・執行／KPI・成果
# 3. 観点別採点          … 5観点×20点の採点表＋総合スコア・総合判定
# 4. 改善提案（詳細）     … 短期／中期／構造見直し（各 効果・障害つき）＋優先打ち手TOP3
# 5. 追加確認事項・根拠箇所 … さらに確認すべき点／根拠となったファイル・ページ
```

### 仕組み（技術メモ）

- フロントエンド `app2/review.htm` が `api2/file-upload.php` にファイルをアップロードし、`api2/review.php` に事業情報＋ファイル参照を JSON で送信します。
- `api2/review.php` はアップロード済みファイルを読み込み、PDF・画像・テキスト系は **inlineData** としてそのまま Gemini に渡し、Office 文書（DOCX 等）は **ZipArchive でテキスト抽出**してから渡します（PDF はブラウザ側 `pdf.js` 抽出を優先し、サーバー側でフォールバック）。
- 生成は Gemini SSE（`streamGenerateContent`）を **NDJSON**（`{"text":...}`）に変換してストリーミング返却します。
- API キーは他アプリと共通の `api/.env` の `GEMINI_API_KEY` を使用します（このアプリ専用の設定は不要）。モデルは `GEMINI_MODEL`（既定 `gemini-3.1-flash-lite`）。生成温度は 0.4。

### 注意事項（行政事業レビューAI 固有）

- 金額・KPI・執行率などの **数値は必ず元資料で確認**してください。AI は推測で数値を補完せず、不明な点は「資料上確認できない」と出力する設計ですが、最終確認は担当部署で行ってください。
- 採点・総合判定はあくまで **補助分析**です。意思決定の根拠とする場合は、レビューシート原本・関係課の確認を経てください。
- スキャンしただけの画像 PDF はテキスト抽出に失敗する場合があります。可能であればテキスト付き PDF または Word を使用してください。
- 機密性の高い情報（機密性3情報等）の入力は避けてください。

## Issue / Pull Request の対応方針

本リポジトリでは、サービスの安定運用に影響する致命的な問題に限り、Issue での報告を受け付けています。Pull Request は受け付けておりません。

### Issues

#### 報告対象となるもの

- データの損失・破損 を引き起こす不具合
- サービスが利用不能になる 障害
- 法令・規則への違反 に関わる問題（例：個人情報の意図しない露出）

#### 報告対象外のもの

以下については、Issue での報告はご遠慮ください。  
テンプレートに合致しない Issue はクローズさせていただく場合があります。

- 機能追加の要望・提案
- 軽微な表示崩れ・誤字脱字
- パフォーマンスの改善提案
- コーディングスタイルに関する指摘
- 質問・使い方の相談

### 対応について

- Issue への対応は、内部の優先度判断に基づき行います
- すべての Issue に対応できるとは限りません
- 対応状況についてのお問い合わせへの個別回答は行っておりません
- 致命的と判断された問題については、可能な範囲で対応状況を Issue 上でお知らせします
